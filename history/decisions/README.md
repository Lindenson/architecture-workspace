# History — Decisions

> STATUS: starter. This is the **project memory of WHY** decisions were made —
> the narrative record complementing the formal [ADRs](../../architecture/adr/README.md).
> ADRs say *what* we decided; this captures the *story, options and trade-offs*
> over time. Closely related to [`../../project-memory/`](../../project-memory/README.md).

## When to add a decision record

Any significant decision that shaped the system — including ones that did **not**
become an ADR (process choices, team agreements, deferred options). Use
[`_TEMPLATE-decision.md`](_TEMPLATE-decision.md).

## Relationship to other records

- Formal architecture decision → [ADR](../../architecture/adr/README.md).
- Rejected option / "why not X" → [project-memory](../../project-memory/README.md).
- Significant operational failure → [incidents](../incidents/README.md).
- What we learned afterward → [lessons-learned](../lessons-learned/README.md).
