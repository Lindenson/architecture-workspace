# History — Incidents

> STATUS: starter. Post-incident records: what broke, impact, root cause and
> follow-ups. Feeds [delivery metrics](../../delivery/metrics/README.md)
> (change-failure rate, MTTR) and often spawns [tech debt](../../quality/technical-debt/README.md)
> or [risks](../../quality/risks/README.md).

## Format

One file per incident, `YYYY-MM-DD-short-title.md`, from
[`_TEMPLATE-incident.md`](_TEMPLATE-incident.md). Blameless: focus on systems
and process, not individuals.

## Relationship to other records

- Architectural weakness exposed → [risk](../../quality/risks/README.md) / [tech debt](../../quality/technical-debt/README.md).
- Lessons to institutionalize → [lessons-learned](../lessons-learned/README.md).
